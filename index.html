Six constitutions for the way AI companies actually grow

Six mental models, each one backed by empirical evidence — never a hot take. If a tactic doesn't trace back to one of these, it doesn't belong in this repo.

  • Living ICP. Drifts because foundation models drift. Weekly revision with evolution log.
  • Technical Credibility Ladder. Five rungs of proof an engineer can verify in five minutes.
  • Community → Customer Funnel. GitHub star → Discord → trial → paid → enterprise. Measurable.
  • Capability Obsolescence Radar. Your real competitor is GPT-6.
  • Integration-Led Growth. Marketing doesn't compound; integrations do.
  • Agent-Native Distribution. If you're not callable by Claude or Cursor, you're invisible.

How it works

Claude reads the repo. You stop pasting context.

Clone the repo. Fill in context/ once. Run GTM tasks from a single prompt. The institutional knowledge persists in git, grows weekly, and arrives context-rich into every Claude Code session.

  • CLAUDE.md — the 2-minute lobby Claude reads first.
  • context/ — your ICP, signals, positioning, competitors, personas. Weekly-revised.
  • skills/ — seven repeatable SKILL.md actions including setup, technical-poc-brief, weekly-drift-audit.
  • research/ — 950 lines of empirical evidence backing every constitution.
  • workflows/ + playbooks/ — decision trees and emergency response scripts.
  • sync/ — MCP-first live data connectors.
