# assets/ · Visual + branded outputs

> Decks, one-pagers, logos, illustrations. Anything a seller, designer, or content team grabs.

## Read first

| File | Purpose |
|---|---|
| **[`design.md`](./design.md)** | Brand visual rules: color, type, spacing, motion. **Open this first.** Every asset must obey these. |
| [`deck-style.md`](./deck-style.md) | PPTX / Keynote template rules. Slide patterns, master layouts, do/don't. |

## Folders

| Folder | Contains | Generated by |
|---|---|---|
| [`decks/`](./decks/) | Pitch / sales / partner decks (.pptx, .key, .pdf) | `skills/deck-build` |
| [`one-pagers/`](./one-pagers/) | Leave-behind PDFs, ICP-targeted variants | `skills/one-pager-build` |
| [`logos/`](./logos/) | Logo files (SVG primary, PNG export) | manual |
| [`illustrations/`](./illustrations/) | Reusable illustration assets | manual + AI gen |

## How Claude helps here

| What you need | Skill to run |
|---|---|
| Build a deck from a prompt | `skills/deck-build` |
| Build a one-pager for a specific ICP | `skills/one-pager-build` |
| (Future: illustration generation, logo variants) | manual for now |

## Workflow

1. Read `design.md` once — internalize the rules
2. For a new deck, run `skills/deck-build` with your prompt + target audience
3. Output lands in `decks/YYYY-MM-DD-{audience}-{purpose}.{ext}`
4. Polish in Keynote / Figma if needed
5. Log in `decks/README.md` (the deck library)

## Anti-patterns

- **Don't use stock illustrations.** They tell the buyer you didn't care.
- **Don't ship a deck without reading `design.md`.** Inconsistent slides ruin a brand faster than bad logos.
- **Don't keep dead asset variants.** If a one-pager hasn't shipped in 6 months, archive it.

---

> **[PROTOCOL]**: `design.md` is binding. When it changes, audit existing assets — old variants get an "outdated" prefix until refreshed.
